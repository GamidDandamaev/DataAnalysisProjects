# Drawdown Risk Prediction from OHLCV Data

## Цель проекта

Проект посвящен прогнозированию риска ближайшей просадки акции AMD.
(AMD) на основе исторических OHLCV-данных.

В отличие от задачи прогноза цены, здесь модель решает задачу бинарной
классификации: определить, будет ли значимая просадка в следующие 5 торговых
дней.

Проект выполнен как учебный ML-проект на финансовых временных рядах.

## Что такое drawdown

Drawdown, или просадка, — это снижение цены актива относительно предыдущего
локального максимума.

Например, если акция стоила 200 долларов, а затем упала до 185 долларов, то
просадка составляет -7.5%.

В проекте просадка используется как показатель рыночного риска.

## Данные

- Источник: Yahoo Finance
- Тикер: AMD
- Актив: AMD.
- Период: 2015-01-01 — 2024-12-31
- Данные: Open, High, Low, Close, Volume

## Целевая переменная

Целевая переменная показывает, возникнет ли значимая просадка в ближайшие
5 торговых дней.

- Target = 1 — в следующие 5 торговых дней цена упадет более чем на 3%
- Target = 0 — такой просадки не будет

Дополнительно рассчитывается futureMaxDrawdown — числовая величина будущей
максимальной просадки. Она используется только для анализа и построения target,
но не используется как признак модели.

## Используемые признаки

В проекте используются признаки, рассчитанные на основе прошлых OHLCV-данных:

| Признак | Описание |
|---|---|
| retLag1 — retLag5 | Лаги логарифмических доходностей |
| rollingRet5 / 10 / 21 | Скользящая доходность за 5, 10 и 21 день |
| rollingVol5 / 10 / 21 | Скользящая волатильность за 5, 10 и 21 день |
| drawdown | Текущая просадка от локального максимума |
| distFromMa50 | Отклонение цены от 50-дневной средней |
| highLowRange | Дневной диапазон High-Low |
| volumeChange | Изменение объема торгов |
| volumeZscore | Z-score объема |
| rsi | Индикатор RSI |

Все признаки строятся только на основе прошлых данных, чтобы не использовать
информацию из будущего.

## Модели

В проекте сравниваются три модели классификации:

| Модель | Тип |
|---|---|
| Logistic Regression | Линейная модель |
| Random Forest | Ансамбль деревьев |
| Gradient Boosting | Градиентный бустинг |

## Валидация

Для проверки качества используется TimeSeriesSplit.

Это означает, что данные не перемешиваются случайным образом. Модель обучается
на более раннем периоде и проверяется на более позднем периоде. Такой подход
лучше подходит для финансовых временных рядов и помогает избежать data leakage.

## Метрики качества

Для оценки моделей используются следующие метрики:

| Метрика | Описание |
|---|---|
| Accuracy | Доля правильных ответов |
| Precision | Доля верных сигналов среди всех сигналов модели |
| Recall | Доля найденных просадок среди всех реальных просадок |
| F1-score | Баланс между precision и recall |
| ROC-AUC | Способность модели разделять классы |
| Confusion Matrix | Матрица ошибок классификации |

## Результаты

Итоговые результаты эксперимента:

| Модель | Accuracy | Precision | Recall | F1 | ROC-AUC |
|---|---:|---:|---:|---:|---:|
| Logistic Regression | 0.5210 | 0.5493 | 0.5781 | 0.5505 | 0.5044 |
| Random Forest | 0.5141 | 0.5368 | 0.6385 | 0.5750 | 0.5163 |
| Gradient Boosting | 0.5127 | 0.5350 | 0.6233 | 0.5673 | 0.5133 |

Качество моделей получилось слабым. Все значения ROC-AUC находятся около 0.5,
то есть модели работают близко к случайному угадыванию.

Формально лучший результат по ROC-AUC показала модель Random Forest, но ее
преимущество над остальными моделями небольшое. Это значит, что по текущему
набору OHLCV-признаков модель плохо отличает периоды с риском просадки от
обычных периодов.

## Основной вывод

Проект показывает, что прогнозирование будущих просадок по одним только
историческим OHLCV-данным является сложной задачей.

Полученные результаты не стоит воспринимать как готовую торговую или
инвестиционную модель. Главная ценность проекта — корректная постановка задачи:
расчет target, создание признаков, временная валидация и честное сравнение
нескольких ML-моделей.


Проект является учебным и имеет несколько ограничений:

- используется только один актив;
- используются только OHLCV-данные;
- гиперпараметры моделей почти не подбирались;
- не учитываются новости, VIX, макроэкономические данные и рыночные индексы;
- качество моделей получилось близким к случайному угадыванию.

Возможные улучшения:

- добавить больше тикеров;
- изменить горизонт прогноза и порог просадки;
- добавить VIX или индекс S&P 500 как внешний фактор;
- использовать балансировку классов;
- провести подбор гиперпараметров;
- попробовать более устойчивые признаки рыночного режима.

Результаты проекта не являются инвестиционной рекомендацией.
